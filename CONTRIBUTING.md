# Contributing to Velo

Thank you for your interest in contributing to Velo!

## Getting Started

1. Read [docs/contributing/local-setup.md](docs/contributing/local-setup.md) to set up your dev environment
2. Check the [open issues](https://github.com/divyeshg94/velo/issues) for good first contributions
3. For large changes, open an issue first to discuss the approach

## Submitting a Pull Request

1. Fork the repo and create a branch: `git checkout -b feature/your-feature`
2. Make your changes following the [coding standards](docs/contributing/coding-standards.md)
3. Add or update tests as needed (see [testing guide](docs/contributing/testing-guide.md))
4. Ensure CI passes: `dotnet test` and `npm test`
5. Open a PR against `main` with a clear description of what and why

## Code of Conduct

Be respectful and constructive. We're building something useful for the Azure DevOps community.
